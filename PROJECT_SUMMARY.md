| 优先级 | 功能 | 原因 |

| **P1** | 心理健康支持 | 40%+ 孩子需要 |
| **P1** | 季节性疾病提醒 | 健康风险 |

### 核心功能流程

```
用户打开小程序
    ↓
授权登录 + 填写健康问卷
    ↓
点击"生成今日计划"
    ↓
获取实时天气 + 读取用户档案
    ↓
匹配天气规则 + 筛选活动
    ↓
生成完整计划 (活动 + 饮食 + 护眼)
    ↓
用户确认 → 执行 → 打卡
    ↓
获得成就 + 数据记录
```

## 🏗️ 技术方案

### 技术栈

| 层级 | 技术 | 成本 |
|------|------|------|
| 前端 | 微信小程序 + Taro | 免费 |
| 后端 | Node.js + NestJS | 免费 |
| 数据库 | PostgreSQL + Redis | 免费 |
| 云服务 | 腾讯云 Lighthouse | ~100 元/月 |
| 天气 API | 和风天气 | 免费额度 |

### 核心服务

- **天气服务**: 实时天气 + 预报 + 预警，30 分钟刷新
- **推荐引擎**: 规则引擎 (MVP) → 机器学习 (后期)
- **用户服务**: 档案管理 + 健康问卷 + 偏好设置
- **打卡服务**: 手动/拍照/视频/语音打卡
- **通知服务**: 模板消息 + 紧急提醒

## 💡 核心洞察 (28 轮迭代后)

### 产品层面

1. **天气适配是最大差异化** - 所有竞品都没有，是核心卖点
2. **运动 + 护眼组合是刚需** - 家长两大焦虑，一个 App 解决
3. **公平性设计至关重要** - 免费活动 50%+，避免经济歧视
4. **安全是第一位的** - 极端天气/运动伤害/法律保障必须覆盖

### 公益层面

1. **公益不是简单的"免费"** - 需要公平性、包容性、可持续性设计
2. **农村市场是蓝海** - 竞品都是城市导向，我们专注普惠
3. **特殊儿童需要关怀** - 残障/慢性病儿童专属方案
4. **透明运营建立信任** - 影响力数据公开，便于获得资助

### 运营层面

1. **学校是最佳落地场景** - 批量覆盖，老师背书
2. **口碑传播最有效** - 真实效果说话，非广告
3. **多元化资金来源** - 不依赖单一来源，降低风险
4. **志愿者是重要力量** - 技术/内容/专家都可以公益参与

### 长期层面

1. **习惯养成是终极目标** - 6 个月周期设计，培养终身运动习惯
2. **超龄过渡需要提前规划** - 18 岁后推荐成人平台
3. **项目停止预案不可少** - 数据导出 + 开源延续
4. **88% 覆盖度已足够** - 无法 100% 覆盖，MVP 可以启动
